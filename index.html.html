Source Database

  • Northwind database was used as the source database
  • The database was created using MySQL
  • ERD diagram was generated using databse reverse engineering

Data Modeling

  • Data Vault method is used to design data warehouse
  • The method is based on normalization and separation of classes of data
  • There are three basic components in a data vault model :
    • Hub - represents the business/surrogate key and separates them from the rest of the model
    • Link - represents the relationship and associations that exist across the business keys
    • Satellites - represents the attributes of business keys and relationships.
  • The process of data modeling using data vault method includes
    • Identifying the hubs
    • Identifying the links
    • Identifying the satellites

Target Database

  • The target data warehouse was created in PostgreSQL

    ETL (Extract, Transform, Load) Process

    • ETL process was done using Python Pandas
    • The data is being tranfered from source csv files to target data warehouse in PostgreSQL
      • The dataframe was extracted from .csv of source database using python pandas
      • The dataframe was transformed based on the data warehouse table requirement
      • Using psycopg2 driver, the transformed dataframe was loaded into target tables in PostgreSQL
